一、简介

在使用OpenAPI进行开发和集成时，遇到错误码是在所难免的，为了帮助开发者们更高效地识别和解决这些问题，我们特别整理了一份详尽的OpenAPI错误码清单，以及针对常见错误码的解决方案。

三、常见错误码解决方案

3.1  解码token异常[网关] 或 不存在应用信息[网关]

正确添加x-acgw-identity，且已针对API进行授权，但在调用时出现：

{
    "errorCode": "2501",
    "messagecn": "不存在应用信息[网关]"
}

或

{
    "errorCode": "2551",
    "messagecn": "解码token异常[网关]"
}

该原因是由于x-acgw-identity不完整导致，请入参完整的x-acgw-identity值后再次发起调用。

x-acgw-identity示例：

djF8MTkyMmQ3YmI5ZDkwMzFlMjE1MDF8NDg4MTI2NDExOTk0MHw6V2LYE7CV5A6x4MzeVf1JgmRVuPMHnuyDv2ZiJ-mVx3w=

3.2 路由未找到[网关] 或 应用未订阅此API[网关]

当添加x-acgw-identity参数发起API请求后，若出现以下错误，是由于未在第三方应用中针对API进行授权；

  • 原厂标准API请通过服务授权，将API授权给第三方应用；

  • 项目二开API，请通过自建API授权，将API授权给第三方应用；

需通过<服务授权/自建API授权>对所使用的API进行访问授权后第三方应用才可正常访问相关API，授权操作是配置可访问的接口范围。未授权的情况下会出现调用失败情况：

1. 服务授权：针对原厂标准API所组成的服务包进行授权，服务包有任何更新时可无感知实时更新权限；

2. 自建API授权：针对项目二开自建API进行授权；

3.3 获取appToken时，报错提示：<title>404 Not Foundtitle>

原因：请求地址错误，检查url末尾是否有空格、换行，或者请求地址多了ierp/等

3.4 获取accessToken时，报错提示："非法操作。","errorCode": "login.loginBizException"

原因：请求方式错误，login.do接口的请求方式必须为“POST”，否则执行失败。

3.5. 未经授权的访问

  • 原因1：accessToken已失效，建议重新获取accessToken。

  • 原因2：第三方应用已被禁用，需要去开放平台启用第三方应用。

3.6. 当前第三方应用不在启用时间范围内

  • 原因：第三方应用当前不可用，需要修改第三方应用的启用时间或停止时间。

3.7. 请求被参数流控限制[网关]

  • 原因：每秒请求大于当前环境的QPS、QPM、QPS、QPD阈值，触发限流阈值；

  • 解决方案：

    1. 等待下一个周期进行限流解除；

    2. 增购专属空间，并提出限流阈值调整申请；

  • 常见问题1：为什么权益统计还有剩余用量，也会被限流拦截；

  • 官方回复1：权益统计所展示的次数，为用户订阅期内的可调用次数，与限流熔断阈值无直接关系；

    1. 限流熔断控制：目前有4个维度的限流，秒级、分钟级、小时级、天级四个维度的调用量分别独立计数，任一维度达到对应阈值即触发该级别限流；

    2. 权益次数控制：取决于客户所购买的API调用量，订阅期内调用量超了且在规定时间内未增购，则所有API调用会被拦截禁止调用，出现请求被计费策略拦截限制[网关]响应。

  • 常见问题2：确认是因触发了QPD的限流阈值引起的请求限制，是否可以申请调整阈值；

  • 官方回复2：标准租户空间，非特殊情况不允许进行阈值调整，这是系统级的限制，针对标准空间进行阈值的调整是存在性能及OOM风险的，正常情况下如果订阅期内日调用峰值会超出阈值，需要购买专属空间。

3.8. 请求被计费策略拦截限制[网关]

  • 原因：订阅期内API调用总次数超出环境所购买的API用量，且超量调用了30天以上，触发了计费策略的拦截；

  • 解决方案：购买用量增购包，进行API用量增购；
